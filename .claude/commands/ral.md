/ralph-loop:ralph-loop Этот проект - порт C++ библиотеки Alembic на Rust.

Нужно проверить совпадение всех важных функций API, Python API.
Нужно корректным путём реализовать все функции, консультируясь с референсным кодом в _ref
Найти все нелогичные места и возможности для оптимизации, в том числе Rust-specific.
Где можно используй готовые крейты если они подходящего качества.
У нас есть улучшения по сравнение с референсом, типа вьювера и всего такого - их тоже нужно отполировать и довести до высокого продакшн качества.
Используй sequential thinking and memory mcp, используй filesystem mcp, в filesystem есть множество инструментов.
Сделай для себя FINDINGS.md и следуй ему, помечая галочками сделанное.
Не пропускай ничего. Do not assume, do not guess, check everything.

Особое внимание уделить совместимости наших файлов и кода с оригинальной библиотекой: нам нужна буквально бинарная совместимость.
Все хэши должны совпадать.
Тесты должны обеспечивать полное покрытие, все edge cases.
Временные файлы класть в test_output/, он у нас в .gitignore.

Viewer очень сильно тормозит: каждые несколько секунд происходит "рывок" производительности.
Плюс само качество рендера довольно низкое.
Нужно разработать унифицированное решение которое сможет корректно показывать геометрию с HDR освещением, с корректной прозрачностью,
с материалами (Autodesk Standard Surface support) и с path tracing. Нужно изучить насколько вообще возможно сделать вьювер с path tracing для работы сразу на всех трёх платформах windows, linux и macos.
Можно кардинально менять архитектуру, но нужно сделать лучше. Всегда составляй для себя PLAN_N.md (где N - следующая доступная цифра) и следуй ему, отмечая галочками сделанное.
В коде всегда оставляй rustdocs и комментарии, где в нужных местах обьясняй что это, где используется, почему сделано именно так и что использует.
Записывай свои находки в файл FINDINGS.md и помечай сделанное галочками чтобы не делать одно и то же по сто раз.
